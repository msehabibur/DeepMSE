Graph Theory Overview

Graph theory is a field within mathematics and computer science that involves the study of graphs, which are abstract representations of a set of objects where some pairs of the objects are connected by links. The interconnected objects are typically represented by points termed as vertices (or nodes), and the links that connect some pairs of vertices are called edges.

Key Concepts

  • Vertex (Node): A fundamental unit in a graph, representing an entity or object.
  • Edge: A connection between two vertices. Edges can be either undirected (bidirectional) or directed (having a specific direction).
  • Weight: A numerical value assigned to an edge (or sometimes a node) that often represents the strength or capacity of the connection.
  • Path: A sequence of edges that connects a sequence of vertices.
  • Graph Types: Includes undirected, directed, weighted, unweighted, and others.

Applications of Graph Theory

Graph theory is used in various fields including computer science, biology, sociology, transportation, and more. Applications include network analysis, circuit design, scheduling, data organization, and many others.

Graph Neural Networks (GNNs)

Graph Neural Networks (GNNs) are a class of neural networks designed to operate on data represented in graph form. They are particularly effective for learning from data where the relationships and interactions between entities are as important as the entities themselves. GNN represent a significant advancement in the field of deep learning, enabling the analysis and prediction of complex relationships in structured data.

Key Characteristics of GNNs

  • Node Representation Learning: GNNs learn to represent nodes (or vertices) in a way that captures both their features and their context within the graph.
  • Edge Representation: They also take into account the edges (connections) between nodes, which can be directed, undirected, and sometimes weighted.
  • Message Passing: GNNs use a technique called message passing, where nodes aggregate information from their neighbors, effectively allowing them to 'communicate' with each other.

Applications of GNNs

GNNs are used in various domains, including:

  • Social network analysis
  • Recommendation systems
  • Drug discovery
  • Fraud detection
  • Autonomous driving

Types of GNNs

Several variants of GNNs exist, each suited to different kinds of graph data and tasks:

  • Graph Convolutional Networks (GCNs): Ideal for semi-supervised learning on graph-structured data.
  • Graph Attention Networks (GATs): They assign varying levels of importance to nodes' neighbors.
  • GraphSAGE: Intended for inductive learning tasks on large graphs.

Cora Dataset Summary¶

The Cora dataset is a cornerstone in graph machine learning, representing a citation network where nodes are scientific papers and edges denote citations, typically in a directed manner. Each paper is characterized by a sparse, high-dimensional feature vector derived from a bag-of-words representation of its text, and is classified into one of several subject areas, making it ideal for node classification tasks in GNNs. The dataset poses challenges typical of real-world graph data, such as complex node interconnections and high-dimensional features, providing invaluable practical experience in handling graph-structured data and understanding the intricacies of GNNs.

import torch
import torch.nn as nn
import torch.nn.functional as F
from torch_geometric.datasets import Planetoid
from torch_geometric.transforms import NormalizeFeatures
from torch.utils.data import DataLoader, TensorDataset

# Load the Cora dataset
dataset = Planetoid(root=".", name="Cora", transform=NormalizeFeatures())
data = dataset[0]

# Prepare the dataset
X = data.x
y = data.y

# Splitting the data into train and test sets
train_mask = data.train_mask
test_mask = data.test_mask

train_dataset = TensorDataset(X[train_mask], y[train_mask])
test_dataset = TensorDataset(X[test_mask], y[test_mask])

train_loader = DataLoader(train_dataset, batch_size=64, shuffle=True)
test_loader = DataLoader(test_dataset, batch_size=64, shuffle=False)

# Define a simple feedforward neural network
class VanillaNN(nn.Module):
    def __init__(self, input_dim, output_dim):
        super(VanillaNN, self).__init__()
        self.fc1 = nn.Linear(input_dim, 64)
        self.fc2 = nn.Linear(64, output_dim)

    def forward(self, x):
        x = F.relu(self.fc1(x))
        x = self.fc2(x)
        return F.log_softmax(x, dim=1)

# Instantiate the model
model = VanillaNN(dataset.num_node_features, dataset.num_classes)

# Loss and optimizer
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=0.01)

# Training the model
def train(epoch):
    model.train()
    for data, target in train_loader:
        optimizer.zero_grad()
        output = model(data)
        loss = criterion(output, target)
        loss.backward()
        optimizer.step()
    print(f'Epoch {epoch}: Training Loss: {loss.item():.4f}')

for epoch in range(1, 51):
    train(epoch)

# Testing the model
def test():
    model.eval()
    correct = 0
    with torch.no_grad():
        for data, target in test_loader:
            output = model(data)
            pred = output.max(1, keepdim=True)[1]
            correct += pred.eq(target.view_as(pred)).sum().item()

    print(f'Test Accuracy: {100. * correct / len(test_loader.dataset):.2f}%')

test()

Adjacency Matrix Overview

The adjacency matrix is a key concept in graph theory, used to represent the connections between nodes in a graph. It's a square matrix that serves as a core data structure in both theoretical and computational analyses of graphs.

Basic Definition

  • Nodes and Edges: In a graph with N nodes, the edges represent connections between these nodes.
  • Matrix Representation: The adjacency matrix for this graph is an N x N matrix, with each element indicating the presence or absence of an edge between two nodes.

Elements of the Matrix

  • Element Value: Typically contains 1s and 0s, where 1 indicates an edge between two nodes, and 0 indicates no edge.
  • Diagonal Elements: These elements usually have a value of 0, representing that nodes do not connect to themselves.

Variations

  • Undirected Graphs: The adjacency matrix is symmetric, indicating mutual connections.
  • Directed Graphs: The matrix is not necessarily symmetric, reflecting the direction of connections.
  • Weighted Graphs: Contains edge weights instead of just 1s, representing the strength or cost of connections.

Example

Consider a graph with three nodes A, B, and C, where A is connected to B, and B to C. The adjacency matrix would be:

  A B C
A 0 1 0
B 1 0 1
C 0 1 0

Applications

Adjacency matrices are used in graph algorithms, network analysis, social network modeling, and as a fundamental tool in GNNs among other applications in computer science and mathematics. In summary, the adjacency matrix is a versatile tool for representing and analyzing graphs, with applications ranging from mathematics to computer science and machine learning.

Building a Neural Network with the Cora Dataset Using Adjacency Matrix

When building a neural network with the Cora dataset, incorporating the graph's adjacency matrix is essential to account for the relationships between nodes. Unlike approaches that treat each node as an independent data point, this method leverages the connections (edges) represented by the adjacency matrix to enhance prediction accuracy in graph-based learning contexts.

Importance of the Adjacency Matrix

The adjacency matrix is crucial in graph-based neural network models as it captures the essential structure of the graph:

  • Each element in the matrix indicates whether a pair of nodes is connected, thereby representing the edges in the graph.
  • This matrix is key to understanding the relationships and interactions between nodes, which are vital in tasks like node classification.

Advantages of Utilizing the Adjacency Matrix

Integrating the adjacency matrix into the neural network model provides several advantages:

  • It allows the model to consider not just the features of individual nodes, but also how nodes are interconnected.
  • This approach is particularly beneficial for tasks where the node's context within the graph (its connections to other nodes) significantly influences its properties or classification.

Overall, using the adjacency matrix in neural network models with the Cora dataset (or similar graph-structured data) facilitates a more nuanced and context-aware analysis, leading to potentially more accurate and insightful results.

Training the Cora Dataset with Graph Convolutional Operations

Training the Cora dataset using graph convolutional operations involves utilizing the specialized techniques of GNNs, specifically Graph Convolutional Networks (GCNs), to process and learn from graph-structured data.

Understanding Graph Convolutional Networks

Graph Convolutional Networks are a type of GNN tailored for processing data represented in graph form. They are particularly effective for tasks like node classification in graphs:

  • Graph Convolution: This operation extends the concept of convolution from traditional neural networks to graph data, allowing the model to aggregate and transform feature information from a node's neighbors.
  • Message Passing: In GCNs, nodes 'communicate' with each other through a process called message passing, where each node updates its features based on its own features and those of its neighbors.

import torch
import torch.nn.functional as F
from torch_geometric.nn import GCNConv
from torch_geometric.datasets import Planetoid
from torch_geometric.transforms import NormalizeFeatures

# Load the Cora dataset
dataset = Planetoid(root=".", name="Cora", transform=NormalizeFeatures())
data = dataset[0]  # The graph data object contains the data of the Cora dataset

# Define the GCN model
class GCN(torch.nn.Module):
    def __init__(self, input_dim, hidden_dim, output_dim):
        super(GCN, self).__init__()
        self.conv1 = GCNConv(input_dim, hidden_dim)  # First GCN layer
        self.conv2 = GCNConv(hidden_dim, output_dim)  # Second GCN layer

    def forward(self, data):
        x, edge_index = data.x, data.edge_index  # Extract node features and edge indices

        # Forward pass through the first GCN layer
        x = self.conv1(x, edge_index)
        x = F.relu(x)  # Apply ReLU activation function
        x = F.dropout(x, training=self.training)  # Apply dropout for regularization
        x = self.conv2(x, edge_index)  # Forward pass through the second GCN layer

        return F.log_softmax(x, dim=1)  # Apply log softmax to output for classification

# Instantiate the model
model = GCN(dataset.num_node_features, 16, dataset.num_classes)  # Initialize the GCN model

# Loss and optimizer
criterion = torch.nn.CrossEntropyLoss()  # Define the loss function
optimizer = torch.optim.Adam(model.parameters(), lr=0.01)  # Define the optimizer

# Training the model
def train(data):
    model.train()  # Set the model to training mode
    optimizer.zero_grad()  # Clear gradients
    out = model(data)  # Forward pass
    loss = criterion(out[data.train_mask], data.y[data.train_mask])  # Compute loss
    loss.backward()  # Backpropagate the loss
    optimizer.step()  # Update model parameters
    return loss.item()  # Return the loss value


def test_model(data):
    model.eval()  # Set the model to evaluation mode
    test_loss = 0
    correct = 0
    total = 0

    with torch.no_grad():  # No gradients needed for evaluation
        for mask_type in ['test_mask']:  # We need only the test mask here
            mask = getattr(data, mask_type)
            logits = model(data)[mask]  # Get predictions for test data
            labels = data.y[mask]
            test_loss += criterion(logits, labels).item()  # Sum up batch loss
            _, predicted = torch.max(logits, 1)
            total += labels.size(0)
            correct += predicted.eq(labels).sum().item()

    test_loss /= total  # Average loss
    test_accuracy = 100. * correct / total  # Calculate accuracy
    test_rmse = torch.sqrt(torch.tensor(test_loss))  # Calculate RMSE

    return test_loss, test_accuracy, test_rmse.item()  # Return loss, accuracy, and RMSE

# Main training loop
for epoch in range(1, 201):
    train_loss = train(data)  # Train the model for one epoch
    test_loss, test_acc, test_rmse = test_model(data)  # Test the model
    # Print epoch number, training loss, test loss, test accuracy, and test RMSE
    print(f'Epoch: {epoch:03d}, '
          f'Train Loss: {train_loss:.4f}, '
          f'Test Loss: {test_loss:.4f}, '
          f'Test Acc: {test_acc:.4f}, '
          f'Test RMSE: {test_rmse:.4f}')

Training the Cora Dataset with Graph Attention Mechanisms

Training the Cora dataset using graph attention mechanisms leverages the advanced capabilities of Graph Neural Networks (GNNs), specifically Graph Attention Networks (GATs), to process and learn from graph-structured data. GATs enhance the traditional message-passing framework by introducing learnable attention coefficients, which prioritize the most relevant neighbors for each node.

Understanding Graph Attention Networks

Graph Attention Networks are a type of GNN designed to address the limitations of conventional graph convolutional methods by incorporating attention mechanisms. These models are highly effective for tasks like node classification in graphs:

  • Graph Attention: This operation introduces an attention mechanism that assigns learnable weights to edges, allowing the model to focus more on important neighbors while ignoring less relevant ones.
  • Multi-Head Attention: GATs often use multi-head attention, where multiple attention mechanisms operate in parallel to capture diverse relationships in the graph.
  • Message Passing with Attention: Nodes update their features by aggregating information from their neighbors, weighted by the attention coefficients. This process ensures that the most critical relationships are emphasized during training.

import torch
import torch.nn.functional as F
from torch_geometric.nn import GATConv
from torch_geometric.datasets import Planetoid
from torch_geometric.transforms import NormalizeFeatures

# Load the Cora dataset
dataset = Planetoid(root=".", name="Cora", transform=NormalizeFeatures())
data = dataset[0]  # The graph data object contains the data of the Cora dataset

# Define the GAT model
class GAT(torch.nn.Module):
    def __init__(self, input_dim, hidden_dim, output_dim, heads=8):
        super(GAT, self).__init__()
        # First GAT layer with multi-head attention
        self.conv1 = GATConv(input_dim, hidden_dim, heads=heads, dropout=0.6)
        # Second GAT layer with a single output head
        self.conv2 = GATConv(hidden_dim * heads, output_dim, heads=1, concat=False, dropout=0.6)

    def forward(self, data):
        x, edge_index = data.x, data.edge_index  # Extract node features and edge indices

        # Forward pass through the first GAT layer
        x = self.conv1(x, edge_index)
        x = F.elu(x)  # Apply ELU activation function
        x = F.dropout(x, p=0.6, training=self.training)  # Apply dropout for regularization
        x = self.conv2(x, edge_index)  # Forward pass through the second GAT layer

        return F.log_softmax(x, dim=1)  # Apply log softmax to output for classification

# Instantiate the model
model = GAT(dataset.num_node_features, 8, dataset.num_classes, heads=8)  # Initialize the GAT model

# Loss and optimizer
criterion = torch.nn.CrossEntropyLoss()  # Define the loss function
optimizer = torch.optim.Adam(model.parameters(), lr=0.005, weight_decay=5e-4)  # Define the optimizer

# Training the model
def train(data):
    model.train()  # Set the model to training mode
    optimizer.zero_grad()  # Clear gradients
    out = model(data)  # Forward pass
    loss = criterion(out[data.train_mask], data.y[data.train_mask])  # Compute loss
    loss.backward()  # Backpropagate the loss
    optimizer.step()  # Update model parameters
    return loss.item()  # Return the loss value


def test_model(data):
    model.eval()  # Set the model to evaluation mode
    correct = 0
    with torch.no_grad():  # No gradients needed for evaluation
        logits = model(data)
        for mask_type in ['test_mask']:  # We need only the test mask here
            mask = getattr(data, mask_type)
            preds = logits[mask].max(1)[1]  # Get predictions
            correct += preds.eq(data.y[mask]).sum().item()

    accuracy = correct / data.test_mask.sum().item()  # Calculate accuracy
    return accuracy

# Main training loop
for epoch in range(1, 201):
    train_loss = train(data)  # Train the model for one epoch
    test_acc = test_model(data)  # Test the model
    # Print epoch number, training loss, and test accuracy
    print(f'Epoch: {epoch:03d}, '
          f'Train Loss: {train_loss:.4f}, '
          f'Test Acc: {test_acc:.4f}')

Training the Cora Dataset with GraphSAGE

Training the Cora dataset using GraphSAGE (Graph Sample and Aggregate) utilizes the unique inductive capabilities of Graph Neural Networks (GNNs) to process graph-structured data. GraphSAGE focuses on generalizing to unseen nodes or graphs by learning a function to aggregate and transform information from a node's neighborhood.

Understanding GraphSAGE

GraphSAGE is a type of GNN specifically designed to address scalability and inductive learning challenges in large or dynamic graphs. It achieves this through a flexible and efficient message-passing framework:

  • Neighborhood Sampling: GraphSAGE samples a fixed-size set of neighbors for each node, making it computationally efficient and scalable for large graphs.
  • Aggregation Functions: GraphSAGE provides flexibility in how neighbor features are aggregated, with options such as mean, max, LSTM-based, or pooling aggregators. This adaptability allows the model to learn meaningful representations for diverse graph structures.
  • Inductive Learning: Unlike traditional GNNs, GraphSAGE learns a mapping function that can be applied to nodes in previously unseen graphs, making it highly suitable for dynamic or expanding datasets.

import torch
import torch.nn.functional as F
from torch_geometric.nn import SAGEConv
from torch_geometric.datasets import Planetoid
from torch_geometric.transforms import NormalizeFeatures

# Load the Cora dataset
dataset = Planetoid(root=".", name="Cora", transform=NormalizeFeatures())
data = dataset[0]  # The graph data object contains the data of the Cora dataset

# Define the GraphSAGE model
class GraphSAGE(torch.nn.Module):
    def __init__(self, input_dim, hidden_dim, output_dim):
        super(GraphSAGE, self).__init__()
        self.conv1 = SAGEConv(input_dim, hidden_dim)  # First GraphSAGE layer
        self.conv2 = SAGEConv(hidden_dim, output_dim)  # Second GraphSAGE layer

    def forward(self, data):
        x, edge_index = data.x, data.edge_index  # Extract node features and edge indices

        # Forward pass through the first GraphSAGE layer
        x = self.conv1(x, edge_index)
        x = F.relu(x)  # Apply ReLU activation function
        x = F.dropout(x, p=0.5, training=self.training)  # Apply dropout for regularization
        x = self.conv2(x, edge_index)  # Forward pass through the second GraphSAGE layer

        return F.log_softmax(x, dim=1)  # Apply log softmax to output for classification

# Instantiate the model
model = GraphSAGE(dataset.num_node_features, 16, dataset.num_classes)  # Initialize the GraphSAGE model

# Loss and optimizer
criterion = torch.nn.CrossEntropyLoss()  # Define the loss function
optimizer = torch.optim.Adam(model.parameters(), lr=0.01, weight_decay=5e-4)  # Define the optimizer

# Training function
def train(data):
    model.train()  # Set the model to training mode
    optimizer.zero_grad()  # Clear gradients
    out = model(data)  # Forward pass
    loss = criterion(out[data.train_mask], data.y[data.train_mask])  # Compute loss
    loss.backward()  # Backpropagate the loss
    optimizer.step()  # Update model parameters
    return loss.item()  # Return the loss value

# Test function
def test(data):
    model.eval()  # Set the model to evaluation mode
    logits = model(data)  # Get model predictions
    masks = ['train_mask', 'val_mask', 'test_mask']  # Define masks
    accuracies = {}

    with torch.no_grad():  # No gradients needed for evaluation
        for mask_type in masks:
            mask = getattr(data, mask_type)  # Get the appropriate mask
            preds = logits[mask].argmax(dim=1)  # Get predictions for nodes
            correct = preds.eq(data.y[mask]).sum().item()  # Count correct predictions
            accuracies[mask_type] = correct / mask.sum().item()  # Compute accuracy

    return accuracies

# Main training loop
for epoch in range(1, 201):
    train_loss = train(data)  # Train the model for one epoch
    accuracies = test(data)  # Test the model on train, validation, and test sets

    # Print epoch number, training loss, and accuracies
    print(f'Epoch: {epoch:03d}, '
          f'Train Loss: {train_loss:.4f}, '
          f'Train Acc: {accuracies["train_mask"]:.4f}, '
          f'Val Acc: {accuracies["val_mask"]:.4f}, '
          f'Test Acc: {accuracies["test_mask"]:.4f}')

Understanding Equivariant Networks

Equivariant networks are a specialized class of neural networks designed to respect the inherent symmetries in data. These networks ensure that transformations applied to the input produce corresponding transformations in the output, a property known as equivariance.

What is Equivariance?

Equivariance is the idea that the model’s behavior should align with the symmetries of the system it models. Mathematically, a function f is equivariant to a transformation T if:

f(T(x)) = T'(f(x))

Here:

  • T(x) represents a transformation of the input data.
  • T'(y) represents the corresponding transformation of the output.

Applications of Equivariant Networks

Equivariant networks are particularly powerful in domains where symmetries play a critical role:

  • Physics Simulations: Modeling systems that obey physical laws like rotational and translational invariance.
  • Molecular Dynamics: Predicting properties of molecules while respecting 3D transformations like rotations and reflections.
  • Computer Vision: Handling geometric transformations in images.

Equivariant Graph Neural Networks

In the context of Graph Neural Networks (GNNs), equivariance ensures that the model’s predictions are consistent with the graph’s structure and any geometric transformations. Some examples include:

  • SE(3)-Equivariant Networks: These are designed to handle 3D transformations like rotations and translations, commonly used in molecular and physical systems.
  • E(n)-Equivariant Networks: A generalization for transformations in n-dimensional space.
  • Permutation Equivariance: Ensures that node reordering in a graph does not affect the model’s output.

Benefits of Equivariant Networks

  • Data Efficiency: Encodes symmetries directly into the architecture, reducing the need for large datasets.
  • Improved Generalization: Better performance on unseen data due to consistent behavior under transformations.
  • Scientific Applications: Enables accurate modeling of systems governed by physical or geometric laws.

Equivariant networks represent a step forward in designing models that inherently understand the symmetries of the data, making them highly effective for specialized tasks in science and engineering.

How do you define edge features for molecular graphs?

In molecular graphs, edge features capture information about the interactions or bonds between atoms. Common edge features include:

  • Bond type: Single, double, triple, or aromatic bonds, encoded as a one-hot vector.
  • Bond length: The distance between two atoms, normalized to a suitable scale.
  • Bond angle: The angle between two connected bonds, if applicable.
  • Edge weight: A scalar value representing bond strength or interaction energy.

For 3D graphs, continuous edge features like distances are often used with radial basis function (RBF) expansions to make them more expressive for GNNs.

Now Let's Create a Movie Recommendation System Using GCN

In this section, we will build a movie recommendation system using a Graph Convolutional Network (GCN). The dataset we use is the MovieLens dataset, which contains information about users, movies, and their ratings. The goal is to predict whether a user will interact with (rate) a movie, a task commonly referred to as link prediction.

Step 1: Download and Load the Dataset

First, we download the MovieLens dataset, which provides metadata about movies and user ratings. We will process the data and prepare it for graph construction. 

# Import necessary libraries
from torch_geometric.data import download_url, extract_zip

movies_path = './ml-latest-small/movies.csv'

ratings_df = pd.read_csv(ratings_path)  # Ratings data

print("Movies Dataset:")
print(movies_df.head())

print("\nRatings Dataset:")

Step 2: Construct the Graph

Next, we construct a heterogeneous graph using PyTorch Geometric's HeteroData. Users and movies are represented as nodes, while ratings form the edges connecting these nodes. Additionally, we use one-hot encoding for node features since the dataset does not provide pre-defined features. 

# Import necessary libraries for graph construction
import torch
from torch_geometric.data import HeteroData

# Initialize a heterogeneous graph
data = HeteroData()

# Add user nodes (one-hot encoding for features)
data['user'].num_nodes = ratings_df['userId'].max() + 1
data['user'].x = torch.eye(data['user'].num_nodes)

# Add movie nodes (one-hot encoding for features)
data['movie'].num_nodes = movies_df.shape[0]
data['movie'].x = torch.eye(data['movie'].num_nodes)

# Add edges (user -> movie) with ratings as edge attributes
data['user', 'rates', 'movie'].edge_index = torch.tensor(
    [ratings_df['userId'].values, ratings_df['movieId'].values - 1],  # Adjust for 0-indexing
    dtype=torch.long
)
data['user', 'rates', 'movie'].edge_attr = torch.tensor(
    ratings_df['rating'].values, dtype=torch.float32
)

# Display the graph summary
print("\nHeterogeneous Graph Summary:")
print(data)

Step 3: Define the GCN Model

We define a simple Graph Convolutional Network (GCN) model for encoding user and movie nodes. To handle the heterogeneous graph, we use to_hetero, which adapts the GCN model for multiple node and edge types. 

# Import necessary libraries for model definition
import torch.nn as nn
from torch_geometric.nn import SAGEConv, to_hetero

# Define a simple GCN model for encoding nodes
class GNNEncoder(nn.Module):
    def __init__(self, hidden_channels):
        super().__init__()
        self.conv1 = SAGEConv((-1, -1), hidden_channels)
        self.conv2 = SAGEConv((-1, -1), hidden_channels)

    def forward(self, x, edge_index):
        x = self.conv1(x, edge_index)
        x = torch.relu(x)
        x = self.conv2(x, edge_index)
        return x

# Define the full model for link prediction
class LinkPredictionModel(nn.Module):
    def __init__(self, hidden_channels):
        super().__init__()
        self.encoder = GNNEncoder(hidden_channels)
        self.encoder = to_hetero(self.encoder, data.metadata())  # Handle heterogeneous graphs

    def decode(self, z, edge_label_index):
        # Dot product of node embeddings for link prediction
        source = z[edge_label_index[0]]
        target = z[edge_label_index[1]]
        return (source * target).sum(dim=1)

    def forward(self, x_dict, edge_index_dict, edge_label_index):
        z_dict = self.encoder(x_dict, edge_index_dict)
        return self.decode(z_dict['user'], edge_label_index)

Step 4: Train the Model

We train the model using the Binary Cross Entropy (BCE) loss. To handle large graphs, we use the LinkNeighborLoader to sample batches of neighbors during training. 

# Import necessary libraries for training
from torch_geometric.loader import LinkNeighborLoader
from torch.nn.functional import binary_cross_entropy_with_logits

# Prepare data loader
train_loader = LinkNeighborLoader(
    data,
    edge_label_index=('user', 'rates', 'movie'),
    edge_label=data['user', 'rates', 'movie'].edge_attr,
    batch_size=32,  # Smaller batch size to avoid memory issues
    num_neighbors=[5, 5],  # Sample fewer neighbors
    shuffle=True
)

# Initialize model and optimizer
model = LinkPredictionModel(hidden_channels=64)
optimizer = torch.optim.Adam(model.parameters(), lr=0.01)

# Training loop
for epoch in range(10):
    model.train()
    total_loss = 0

    for batch in train_loader:
        optimizer.zero_grad()

        # Forward pass
        pred = model(batch.x_dict, batch.edge_index_dict, batch['user', 'rates', 'movie'].edge_label_index)
        loss = binary_cross_entropy_with_logits(pred, batch['user', 'rates', 'movie'].edge_label)

        # Backward pass
        loss.backward()
        optimizer.step()
        total_loss += loss.item()

    print(f"Epoch {epoch + 1}, Loss: {total_loss:.4f}")

Step 5: Evaluate the Model

Finally, we evaluate the model's performance using the ROC-AUC (Receiver Operating Characteristic - Area Under Curve) metric, which measures the model's ability to predict links accurately. 

# Import necessary libraries for evaluation
from sklearn.metrics import roc_auc_score

model.eval()
with torch.no_grad():
    preds = []
    labels = []

    for batch in train_loader:
        pred = model(batch.x_dict, batch.edge_index_dict, batch['user', 'rates', 'movie'].edge_label_index)
        preds.append(torch.sigmoid(pred).cpu())
        labels.append(batch['user', 'rates', 'movie'].edge_label.cpu())

    preds = torch.cat(preds)
    labels = torch.cat(labels)
    auc = roc_auc_score(labels.numpy(), preds.numpy())

print(f"ROC AUC Score: {auc:.4f}")

Comparison of GCN, GraphSAGE, and Attention Mechanisms

Aspect Graph Convolutional Networks (GCN) GraphSAGE Attention Mechanism (GAT)
Neighborhood Aggregation Aggregates all neighbors equally based on a normalized adjacency matrix. Samples a fixed number of neighbors for aggregation. Assigns different importance (attention scores) to each neighbor during aggregation.
Scalability Not scalable to very large graphs due to dense adjacency matrix operations. Highly scalable as it samples neighbors instead of processing all of them. Scalable but limited by the computational cost of calculating attention scores for all neighbors.
Aggregation Function Weighted sum using adjacency matrix normalization. Flexible: Mean, max, or LSTM-based aggregation. Weighted sum using attention coefficients.
Key Feature Treats all neighbors equally during aggregation. Allows sampling neighbors, making it efficient for large-scale graphs. Dynamically assigns weights to neighbors based on their importance.
Example In a social network, the feature of a person (node) is updated as an average of all their friends' features. For example, Person A with 3 friends will aggregate their features equally. In a social network, GraphSAGE might sample 2 out of 10 neighbors (friends) of Person A. Only those sampled neighbors contribute to the feature update. In a social network, GAT might assign higher weights to closer friends of Person A (based on interaction frequency), making their contribution more significant in the feature update.

1. Graph Convolutional Networks (GCN)

How it Works: GCN aggregates information from all neighbors equally using a normalized adjacency matrix.

Formula:

H(l+1) = σ(D-1/2 A D-1/2 H(l) W(l))
  • A: Adjacency matrix.
  • D: Degree matrix.
  • H(l): Node features at layer l.
  • W(l): Learnable weight matrix.

Example: In a graph of students, if each student has a GPA (node feature), the GPA of a student is updated by averaging the GPAs of all their friends.


from torch_geometric.nn import GCNConv
import torch.nn.functional as F


class GCN(torch.nn.Module):
def init(self, in_channels, out_channels):
super(GCN, self).init()
self.conv1 = GCNConv(in_channels, 16)
self.conv2 = GCNConv(16, out_channels)


def forward(self, x, edge_index):
    x = self.conv1(x, edge_index)
    x = F.relu(x)
    x = self.conv2(x, edge_index)
    return F.log_softmax(x, dim=1)

2. GraphSAGE

How it Works: GraphSAGE samples a fixed number of neighbors for aggregation, making it scalable to large graphs.

Formula:

hv(l+1) = σ(W · Aggregate({hv(l)} ∪ {hu(l) | u ∈ N(v)}))
  • N(v): Neighbors of node v.
  • Aggregation: Mean, max, or LSTM-based aggregation.

Example: In a recommendation system, if a user has rated 100 movies (neighbors), GraphSAGE samples 10 movies randomly and uses their features for aggregation.

3. Attention Mechanism (GAT)

How it Works: GAT assigns attention scores to neighbors, dynamically weighting their contributions based on importance.

Formula:

αij = softmax(LeakyReLU(aT [W hi || W hj]))
  • αij: Attention coefficient for edge (i, j).
  • a: Learnable attention vector.
  • ||: Concatenation operator.

Example: In a molecular graph, GAT assigns higher weights to bonds critical for the molecule’s stability, dynamically prioritizing their importance.


from torch_geometric.nn import GATConv


class GAT(torch.nn.Module):
def init(self, in_channels, out_channels):
super(GAT, self).init()
self.conv1 = GATConv(in_channels, 16, heads=8, concat=True)
self.conv2 = GATConv(16 * 8, out_channels, heads=1, concat=False)


def forward(self, x, edge_index):
    x = self.conv1(x, edge_index)
    x = F.elu(x)
    x = self.conv2(x, edge_index)
    return F.log_softmax(x, dim=1)
