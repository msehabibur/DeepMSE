import numpy as np
import pandas as pd
from sklearn.ensemble import RandomForestRegressor
from sklearn.model_selection import GridSearchCV, train_test_split
from sklearn.metrics import mean_squared_error

# Loading the dataset
data = data = pd.read_csv('/content/drive/MyDrive/DeepMSE/Medical_Insurance_Cost_Prediction_using_Machine_Learning/insurance.csv')

# encoding sex column
data.replace({'sex':{'male':0,'female':1}}, inplace=True)

3 # encoding 'smoker' column
data.replace({'smoker':{'yes':0,'no':1}}, inplace=True)

# encoding 'region' column
data.replace({'region':{'southeast':0,'southwest':1,'northeast':2,'northwest':3}}, inplace=True)

# Splitting the dataset into training and testing sets
X = data.drop('charges', axis=1)  # Features
y = data['charges']  # Target variable
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# Creating a Random Forest Regression model
rf = RandomForestRegressor(random_state=42)

# Defining the hyperparameter grid
param_grid = {
    'n_estimators': [50, 100, 150],
    'max_depth': [None, 5, 10],
    'min_samples_split': [2, 5, 10],
    'min_samples_leaf': [1, 2, 4]
}

# Performing grid search cross-validation
# It specifies the estimator object, which is the model or algorithm that will be optimized during the grid search.
# In this case, the estimator is rf, which is an instance of the RandomForestRegressor.
# It is a dictionary or a list of dictionaries that defines the hyperparameter grid to be searched.
# Each key in the dictionary represents a hyperparameter name, and the corresponding value(s) represent the values to be searched for that hyperparameter.
# This allows us to define multiple combinations of hyperparameters to be evaluated.
# It specifies the number of cross-validation folds to be used during the grid search.
# In this case, cv=5 indicates a 5-fold cross-validation, where the dataset is divided into 5 equal parts, and the model is trained and evaluated 5 times,
# with each part serving as the validation set once.
# It specifies the scoring metric used to evaluate the performance of each combination of hyperparameters.
# In this case, 'neg_mean_squared_error' is used as the scoring metric.
# The negative mean squared error is a common metric used in regression problems, where lower values indicate better performance.
grid_search = GridSearchCV(estimator=rf, param_grid=param_grid, cv=5, scoring='neg_mean_squared_error')
grid_search.fit(X_train, y_train)

# Retrieving the best model
best_model = grid_search.best_estimator_

# Making predictions on the test set using the best model
y_pred = best_model.predict(X_test)

# Evaluating the model's performance
mse = mean_squared_error(y_test, y_pred)
rmse = np.sqrt(mse)
print('Root Mean Squared Error:', rmse)

# Printing the best hyperparameters
print('Best Hyperparameters:', grid_search.best_params_)
# Calculate the feature importances

By performing the grid search cross-validation, the code will evaluate the performance of different combinations of hyperparameters for the RandomForestRegressor model using the specified scoring metric. It will then select the best combination of hyperparameters that yields the lowest mean squared error on average across the cross-validation folds. This helps in finding the optimal set of hyperparameters for the RandomForestRegressor model, improving its performance on unseen data.

import numpy as np
import pandas as pd
from sklearn.ensemble import RandomForestRegressor
from sklearn.model_selection import RandomizedSearchCV, train_test_split
from sklearn.metrics import mean_squared_error
from sklearn.metrics import r2_score

# Loading the dataset
data = pd.read_csv('/content/drive/MyDrive/DeepMSE/Medical_Insurance_Cost_Prediction_using_Machine_Learning/insurance.csv')

# Encoding sex column
data.replace({'sex': {'male': 0, 'female': 1}}, inplace=True)

# Encoding 'smoker' column
data.replace({'smoker': {'yes': 0, 'no': 1}}, inplace=True)

# Encoding 'region' column
data.replace({'region': {'southeast': 0, 'southwest': 1, 'northeast': 2, 'northwest': 3}}, inplace=True)

# Splitting the dataset into training and testing sets
X = data.drop('charges', axis=1)  # Features
y = data['charges']  # Target variable
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# Creating a Random Forest Regression model
rf = RandomForestRegressor(random_state=42)

# Defining the hyperparameter grid
param_grid = {
    'n_estimators': np.arange(50, 151, 50),
    'max_depth': [None, 5, 10],
    'min_samples_split': [2, 5, 10],
    'min_samples_leaf': [1, 2, 4]
}

#n_iter: It specifies the number of parameter settings that are sampled from the specified distributions.
#In other words, it determines how many different combinations of hyperparameters are evaluated during the randomized search.
#In this case, n_iter=10 means that 10 random combinations of hyperparameters will be tested.

# Performing randomized search cross-validation
random_search = RandomizedSearchCV(estimator=rf, param_distributions=param_grid, cv=5, scoring='neg_mean_squared_error', n_iter=10)
random_search.fit(X_train, y_train)

# Retrieving the best model
best_model = random_search.best_estimator_

# Making predictions on the test set using the best model
y_pred = best_model.predict(X_test)

# Evaluating the model's performance
mse = mean_squared_error(y_test, y_pred)
rmse = np.sqrt(mse)
r2 = r2_score(y_test, y_pred)

# Printing the evaluation metrics
print('Root Mean Squared Error:', rmse)
print('R-squared:', r2)

# Printing the best hyperparameters
print('Best Hyperparameters:', random_search.best_params_)

Removing outliers from a ML dataset can help improve the model's performance and prevent it from being overly influenced by extreme values. Here are some common approaches to remove outliers. Statistical methods: One approach is to use statistical methods to identify outliers based on the distribution of the data. You can calculate the z-scores or use the interquartile range (IQR) to detect data points that fall outside a certain threshold. Data points beyond a certain number of standard deviations from the mean or outside a specific range of the IQR can be considered outliers and removed from the dataset. Visualization techniques: Plotting the data can help identify outliers visually. Box plots, scatter plots, and histograms can reveal any data points that deviate significantly from the overall pattern. Observing data points that lie far away from the bulk of the data or exhibit unusual patterns can indicate potential outliers. Domain knowledge: Your knowledge about the domain can guide you in identifying outliers. For example, if you know that a certain feature or target variable should fall within a specific range, any data points outside that range can be considered outliers. Ensemble methods: Ensemble methods, such as Random Forests, are less sensitive to outliers compared to some other models. By using an ensemble model, the impact of outliers can be reduced as the model aggregates predictions from multiple trees. Winsorization or trimming: Instead of removing outliers, you can choose to Winsorize or trim the data. Winsorization involves replacing extreme values with the nearest value within a certain percentile. Trimming simply involves removing the extreme values without replacement. It's important to note that the decision to remove outliers should be made carefully, as outliers may contain valuable information or represent genuine anomalies in the data. It's advisable to analyze the impact of outlier removal on the model's performance and consider the specific context and goals of the analysis before deciding to remove outliers. The Z-score, also known as the standard score, is a statistical measure that quantifies how far away a particular data point is from the mean of a distribution. It is used to normalize and compare data points from different distributions. The formula to calculate the Z-score for a data point x in a distribution with mean μ and standard deviation σ is:

Z = (x - μ) / σ

The Z-score tells us how many standard deviations a data point is away from the mean. A positive Z-score indicates that the data point is above the mean, while a negative Z-score indicates that the data point is below the mean. By calculating the Z-scores for all the data points in a dataset, we can identify outliers. Typically, data points with a Z-score greater than a certain threshold (e.g., 2 or 3) are considered outliers and may be removed from the dataset or treated separately in the analysis. Removing outliers helps in reducing the influence of extreme values on statistical analysis or model fitting.

The difference in performance between a NN model and a RF model can be attributed to several factors. Here are some possible reasons why the NN model may be performing poorly compared to the RF model. Insufficient Training Data: NN typically require a large amount of training data to generalize well. If the dataset used for training the NN model is small, it may lead to overfitting, where the model becomes too specialized to the training data and performs poorly on unseen data. RF, on the other hand, can handle smaller datasets more effectively as they aggregate multiple decision trees. Improper Hyperparameter Tuning: NN have various hyperparameters that need to be tuned, such as the number of hidden layers, the number of neurons in each layer, learning rate, activation functions, etc. If the hyperparameters are not properly tuned, it can lead to suboptimal performance. RF, on the other hand, have fewer hyperparameters to tune, making them relatively easier to optimize. Feature Engineering: NN often require careful feature engineering to extract relevant information from the data. If the input features are not properly preprocessed or selected, it can impact the model's performance. RF, in contrast, are capable of handling raw or less processed data and can automatically learn feature importance. Sensitivity to Initial Conditions: NN are sensitive to initial conditions and can get stuck in local optima during the training process. The model's performance heavily depends on the initialization of the weights and biases. RF, being an ensemble method, are less prone to getting stuck in local optima and can provide more stable predictions. Interpretability: RF offer better interpretability compared to NN. RF provide feature importances, which can help understand the contribution of each feature to the model's predictions. This interpretability aspect can be beneficial for debugging and understanding the model's behavior. It's important to note that the performance of different ML models can vary depending on the dataset and problem at hand. While NN are known for their ability to capture complex patterns in the data, they may not always be the best choice for every problem. Evaluating and comparing the performance of different models on various metrics and understanding the specific characteristics of the dataset are essential steps in choosing the most suitable model.
